492##
493#F  FactorsRho( <n>, <inc>, <cluster>, <limit> )   Pollards rho factorization
494##
495##  `FactorsInt' does trial divisions by the primes less than 1000 to  detect
496##  all composites with a factor less than 1000 and primes less than 1000000.
497##  After that it calls `FactorsRho(<n>,1,16,8192)' to do the hard work.
498##
499##  `FactorsRho'  will  return a  list  of factors   and a list  of composite
500##  number.   Usually  `FactorsInt'  factors  integers  with   prime  factors
501##  $\<1000$ faster.     However  for   integers  with  no   factor  $\<1000$
502##  `FactorsRho' will be faster.
503##
504##  `FactorsRho' uses Pollards $\rho$ method to factor the integer $n = p q$.
505##  For a small simple example lets assume we want to factor $667 = 23 * 29$.
506##  `FactorsRho' first calls `IsPrimeInt' to avoid trying to factor a prime.
507##
508##  Then it uses the sequence defined by  $x_0=1, x_{i+1}=(x_i^2+1)$ mod $n$.
509##  In our example this is $1, 2, 5, 26, 10, 101, 197, 124, 36, 630, .. $.
510##
511##  Modulo $p$ it takes on at most $p-1$ different values, thus it eventually
512##  becomes recurrent, usually this happens after roughly $2 \sqrt{p}$ steps.
513##  In our example modulo 23 we get $1, 2, 5, 3, 10, 9, 13, 9, 13, 9, .. $.
514##
515##  Thus there exist pairs $i, j$ such that $x_i = x_j$ mod $p$,  i.e.,  such
516##  that $p$ divides $Gcd( n, x_j-x_i )$.  With a bit of luck no other factor
517##  of $n$ divides $x_j - x_i$ so we find $p$ if we know such a pair.  In our
518##  example $5, 7$ is the first pair, $x_7-x_5=23$, and $Gcd(667,23) = 23$.
519##
520##  Now it is too expensive to check all pairs, but there also must be  pairs
521##  of the form $2^i-1, j$ with $3*2^{i-1} <= j < 4*2^{i-1}$.  In our example
522##  $7, 13$ is the first such pair, $x_13-x_7=506$, and $Gcd(667,506) = 23$.
523##
524##  Thus by taking the gcds of $n$ and $x_j-x_i$ for such pairs, we will find
525##  the factor $p$ after approximately $2 \sqrt{p} \<= 2 \sqrt^4{n}$ steps.
526##
527##  If $Gcd( n, x_j - x_i )$  is not a prime `FactorsRho'  will  call  itself
528##  recursivly with a different value for <inc>, i.e., it  will try to factor
529##  the gcd using a different sequence $x_{i+1} = (x_i^2 + inc)$ mod $n$.
530##
531##  Since the gcd computations are by far the most time consuming part of the
532##  algorithm  one can save time by  clustering differences and computing the
533##  gcd  only every <cluster>  iteration.  This slightly increases the chance
534##  that a gcd is composite, but reduces the runtime by a large amount.
535##
536##  Finally `FactorsRho' accepts an argument <limit>  which is the number  of
537##  iterations  performed by `FactorsRho' before giving up. The default value
538##  is  8192  which corresponds to a few minutes  while guaranteing that  all
539##  prime factors less than $10^6$ and most less than $10^9$ are found.
540##
541##  Better descriptions of the algorithm and related topics can be found  in:
542##  J. Pollard, A Monte Carlo Method for Factorization, BIT 15, 1975, 331-334
543##  R. Brent, An Improved Monte Carlo Method for Fact., BIT 20, 1980, 176-184
544##  D. Knuth, Seminumerical Algorithms  (TACP II),  AddiWesl,  1973,  369-371
545##
546FactorsRho := function ( n, inc, cluster, limit )
547
548    local   i,  sign,  factors,  composite,  x,  y,  k,  z,  g,  tmp,
549            IsPrimeOrProbablyPrimeInt;
550
551    # make $n$ positive and handle trivial cases
552    sign := 1;
553    if n < 0  then sign := -sign;  n := -n;  fi;
554    if n < 4  then return [ [ sign * n ], [] ];  fi;
555    factors   := [];
556    composite := [];
557    while n mod 2 = 0  do Add( factors, 2 );  n := n / 2;  od;
558    while n mod 3 = 0  do Add( factors, 3 );  n := n / 3;  od;
559
560    if   ValueOption("UseProbabilisticPrimalityTest") = true
561    then IsPrimeOrProbablyPrimeInt := IsProbablyPrimeInt;
562    else IsPrimeOrProbablyPrimeInt := IsPrimeInt; fi;
563
564    if IsPrimeOrProbablyPrimeInt(n)  then Add( factors, n );  n := 1;  fi;
565
566    # initialize $x_0$
567    x := 1;  z := 1;  i := 0;
568
569    # loop until we have factored $n$ completely or run out of patience
570    while 1 < n  and 2^i <= limit  do
571
572        # $y = x_{2^i-1}$
573        y := x;  i := i + 1;
574
575        # $x_{2^i}, .., x_{3*2^{i-1}-1}$ need not be compared to $x_{2^i-1}$
576        for k  in [1..2^(i-1)]  do
577            x := (x^2 + inc) mod n;
578        od;
579
580        # compare $x_{3*2^{i-1}}, .., x_{4*2^{i-1}-1}$ with $x_{2^i-1}$
581        for k  in [1..2^(i-1)]  do
582            x := (x^2 + inc) mod n;
583            z := z * (x - y) mod n;
584
585            # from time to time compute the gcd
586            if k mod cluster = 0  then
587                g := GcdInt( n, z );
588
589                # if it is > 1 we have found a factor which need not be prime
590                if g > 1  then
591                    tmp := FactorsRho(g,inc+1,QuoInt(cluster+1,2),limit);
592                    factors   := Concatenation( factors,   tmp[1] );
593                    composite := Concatenation( composite, tmp[2] );
594
595                    n := n / g;
596                    if IsPrimeOrProbablyPrimeInt(n)  then
597                        Add( factors, n );  n := 1;
598                    fi;
599                fi;
600            fi;
601        od;
602    od;
603
604    # add <n> to the list of composite numbers
605    if 1 < n  then
606        Add( composite, n );
607    fi;
608
609    # sort the list of factors and composite numbers and return it
610    Sort(factors);
611    Sort(composite);
612    if 0 < Length(factors)  then
613        factors[1] := sign * factors[1];
614    else
615        composite[1] := sign * composite[1];
616    fi;
617    return [ factors, composite ];
618
619end;

912#F  IsPrimeInt( <n> ) . . . . . . . . . . . . . . . . . . .  test for a prime
913##
914##  `IsPrimeInt' does trial divisions by the primes less  than 1000 to detect
915##  composites with a factor less than 1000 and  primes  less  than  1000000.
916##
917##  `IsPrimeInt' then checks that $n$ is a strong pseudoprime to the  base 2.
918##  This uses Fermats theorem which says $2^{n-1}=1$ mod $n$ for a prime $n$.
919##  If $2^{n-1}\<>1$ mod $n$, $n$ is composite, `IsPrimeInt' returns `false'.
920##  There are composite numbers for which $2^{n-1}=1$,  but they are  seldom.
921##
922##  Then `IsPrimeInt' checks that $n$ is a Lucas pseudoprime for $p$, choosen
923##  so that the discriminant $d=p^2/4-1$ is an  quadratic nonresidue mod $n$.
924##  I.e., `IsPrimeInt' takes the root $a = p/2+\sqrt{d}$ of $x^2 - px + 1$ in
925##  the  ring $Z_n[\sqrt{d}] and computes the  traces of $a^n$ and $a^{n+1}$.
926##  If $n$ is a prime, this  ring is the field of  order $n^2$ and raising to
927##  the $n$th power is conjugation, so $trace(a^n)=p$ and $trace(a^{n+1})=2$.
928##  However, these identities hold only for extremely few composite numbers.
929##
930##  Note that  this  test  for $trace(a^n) = p$  and  $trace(a^{n+1}) = 2$ is
931##  usually formulated using the Lucas sequences  $U_k = (a^k-b^k)/(a-b)$ and
932##  $V_k = (a^k+b^k)=trace(a^k)$, where one tests $U_{n+1} = 0, V_{n+1} = 2$.
933##  However, the trace test is equivalent and requires fewer multiplications.
934##  Thanks to Daniel R. Grayson (dan@symcom.math.uiuc.edu)  for  telling  me.
935##
936##  `IsPrimeInt' can be shown to return the correct answer for $n < 10^{13}$,
937##  by testing against R.G.E. Pinch's list of all pseudoprimes to base 2 less
938##  than $10^{13}$ ('ftp://dpmms.cam.ac.uk/pub/rgep/PSP/psp13.gz').
939##
940##  Better descriptions of the algorithm and related topics can be found  in:
941##  G. Miller, cf. Algorithms and Complexity ed. Traub, AcademPr, 1976, 35-36
942##  C. Pomerance et.al., Pseudoprimes to 25*10^9, MathComp 35 1980, 1003-1026
943##  D. Knuth, Seminumerical Algorithms  (TACP II),  AddiWesl,  1973,  378-380
944##  G. Gonnet, Heuristic Primality Testing, Maple Newsletter 4,  1989,  36-38
945##  R. Baillie, S. Wagstaff, Lucas Pseudoprimes, MathComp 35 1980,  1391-1417
946##  R. Pinch, Some Primality Testing Algorithms, Notic. AMS 9 1993, 1203-1210
947##

1255    # get the arguments
1256    if   Length(arg) = 1  then n := arg[1];  k := 2;
